# 附件处理

处理用户发送的文件附件和接收的媒体内容。

## 支持的附件类型

### 富媒体接收
- JPEG 图片消息
- PNG 图片消息
- 自动下载到 `~/.openclaw/workspace/media/inbound/` 目录
- 媒体文件统一命名格式：`openclaw-media-{timestamp}.{ext}`

### 文件附件提取
- Word 文档（.docx）- 通过 mammoth 库提取文本内容
- PDF 文档（.pdf）- 通过 pdf-parse 库提取文本内容
- 纯文本文件（.txt、.md、.json、.js、.ts 等）
- 二进制文件（.xlsx、.pptx、.zip 等）- 保存到磁盘并报告路径

## 处理流程

1. 自动检测收到的附件类型
2. 根据文件类型选择合适的解析器
3. 提取内容并注入到 AI 上下文
4. 保存文件到本地存储（如需要）
5. 生成文件访问路径供后续处理

## 代码示例

```javascript
// 接收的附件会自动处理
// AI 可以直接访问附件内容，例如：
// - 图片会自动传递给视觉模型
// - 文档内容会注入到对话上下文
// - 文件路径会在消息中提供

// 在消息处理中，附件内容已经可用
await processMessage({
  text: "请分析这个文档的内容",
  attachments: ["path/to/downloaded/file.pdf"]
});
```

## 配置选项

- `enableMediaUpload`: 启用媒体上传功能
- `mediaMaxMb`: 媒体文件大小限制

## 处理能力

- 自动下载和存储接收到的媒体文件
- 支持多种文档格式的文本提取
- 二进制文件的存储和路径管理
- 图片自动传递给视觉模型进行分析

## 关键要点

- 媒体文件自动下载到指定目录
- 文档内容自动注入到 AI 上下文
- 支持多种文件格式的解析
- 文件大小受到钉钉平台限制
- 视觉模型可以直接处理接收到的图片
- 二进制文件仅保存路径，不解析内容
