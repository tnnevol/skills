# 标准工作流程

每次执行文档操作前，都应遵循以下标准工作流程。

## 工作流程步骤

1. **识别任务类型** → 例如：列知识库、读文档、写文档、创建文档、成员管理
2. **按任务校验所需配置** → 通过辅助脚本读取配置；仅校验本任务必须项
3. **收集缺失配置** → 若缺少某项，一次性询问用户所有缺失值
4. **获取 Token / operatorId** → 直接调用辅助脚本自动处理
5. **执行操作** → 复杂命令写入临时脚本执行

## 配置校验规则

- **所有任务通用必需**：`DINGTALK_APP_KEY`、`DINGTALK_APP_SECRET`、`DINGTALK_MY_USER_ID`
- **涉及任何文档/知识库 API 调用**：必须有 `DINGTALK_MY_OPERATOR_ID`
- **创建/读取/写入/删除/成员管理**：除通用项外，无额外固定配置键

## 特殊注意事项

- 规则：未通过"本次任务配置校验"前，不得进入 API 调用步骤
- 凭证禁止在输出中完整打印，确认时仅显示前 4 位 + `****`
- 凡是包含变量替换、管道或多行逻辑的命令，写入 `/tmp/<task>.sh` 再执行
- 不要把多行命令直接粘到终端里（会被截断）

## 关键要点

- 工作流程确保每次操作都有必要的前置条件
- 配置管理保证安全性
- 临时脚本机制避免复杂命令执行问题
